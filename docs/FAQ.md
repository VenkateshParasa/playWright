## General Questions

### What is the Playwright & Selenium Learning Platform?

The Playwright & Selenium Learning Platform is a comprehensive learning system designed to help developers master browser automation and E2E testing. It combines structured lessons, spaced repetition flashcards, auto-graded exercises, and progress tracking in a Progressive Web App.

### Who is this platform for?

This platform is designed for:
- Developers with basic-medium JavaScript knowledge
- Developers with medium Java knowledge
- QA engineers looking to upskill in automation
- Anyone wanting to learn Playwright and/or Selenium

### How long does it take to complete?

- **30-Day Intensive Track:** 2-3 hours daily (recommended for faster results)
- **60-Day Extended Track:** 1-2 hours daily (more relaxed pace)

Most students complete the program in 30-60 days depending on their chosen track and commitment level.

### Is the platform free?

The core platform is free. Premium features may be available in the future.

### What technologies will I learn?

- **Playwright:** TypeScript/JavaScript for modern browser automation
- **Selenium:** Java for traditional E2E testing
- **Page Object Model:** Design patterns for maintainable tests
- **CI/CD Integration:** Running tests in pipelines
- **Best Practices:** Industry-standard testing approaches

## Getting Started

### How do I get started?

1. Register for an account
2. Verify your email
3. Complete the onboarding wizard
4. Choose your learning track (30-day or 60-day)
5. Start with Day 1!

### Do I need any prior knowledge?

**Prerequisites:**
- Basic JavaScript (variables, functions, async/await)
- Medium Java (OOP concepts, basic syntax)
- Familiarity with HTML and CSS
- Understanding of web browsers

**Not required:**
- No testing experience needed
- No prior automation knowledge needed
- We teach everything from scratch!

### What software do I need to install?

For the platform itself:
- Modern web browser (Chrome, Firefox, Edge, Safari)
- Internet connection (for initial setup)

For hands-on practice (optional but recommended):
- Node.js 20+
- VS Code or your preferred IDE
- Java JDK 17+ (for Selenium exercises)

### Can I switch between 30-day and 60-day tracks?

Yes! You can switch tracks at any time in Settings → Learning Track. Your progress carries over.

## Account & Authentication

### I didn't receive the verification email

Check your spam/junk folder. If you still don't see it:
1. Wait 10 minutes (emails can be delayed)
2. Request a new verification email from the login page
3. Try a different email address
4. Contact support if the issue persists

### How do I reset my password?

1. Click "Forgot Password" on the login page
2. Enter your email address
3. Check your email for reset link
4. Click the link and set a new password
5. Login with your new password

### Can I change my email address?

Yes, in Settings → Profile → Email. You'll need to verify the new email address.

### How do I delete my account?

Go to Settings → Privacy → Delete Account. This is permanent and cannot be undone. All your data will be deleted.

## Flashcards & SRS

### What is Spaced Repetition?

Spaced Repetition is a learning technique that shows you flashcards at optimal intervals to maximize long-term retention. Cards you struggle with appear more frequently; cards you know well appear less often.

### How does the SM-2 algorithm work?

The SM-2 algorithm calculates when to show you each card based on:
- Your quality rating (0-5)
- How many times you've reviewed the card
- The card's ease factor
- Time since last review

You don't need to understand the math - just rate your recall honestly!

### How should I rate my recall?

- **0 - Complete Blackout:** No idea at all
- **1 - Incorrect:** Wrong answer, but I recognized the topic
- **2 - Incorrect (Easy):** Wrong but it felt familiar
- **3 - Correct (Difficult):** Right answer with significant effort
- **4 - Correct (Hesitant):** Right answer with some hesitation
- **5 - Perfect:** Instant, confident recall

Be honest! The algorithm works best with accurate ratings.

### How many cards should I review per day?

Recommended: 20-50 cards per day. You can adjust this in Settings → Study Preferences → Daily Review Limit.

Quality over quantity! Better to do 20 cards well than 100 cards rushed.

### What if I have too many cards due?

- Focus on overdue cards first
- Adjust daily review limit to a manageable number
- Be consistent - review a little every day
- Consider suspending cards you don't need right now

### Can I create my own flashcards?

Yes! Navigate to Flashcards → Create Card. Custom cards follow the same SRS algorithm.

### What happens if I miss a day of reviews?

The cards accumulate and will be due the next day. Try to review daily for best results, but don't stress if you miss a day occasionally.

## Exercises & Coding

### Where do I write code?

Code exercises have a built-in Monaco editor (same engine as VS Code) where you write your solutions directly in the browser.

### Do I need to set up anything locally?

No! All exercises run in the browser or on our servers. However, we recommend practicing locally with the provided code examples for best learning.

### How is my code graded?

Your code is automatically tested against multiple test cases:
- **Visible tests:** You can see these inputs/outputs
- **Hidden tests:** Run after submission to prevent hard-coding

Grading considers:
- Correctness (all tests pass)
- Code quality (readable, follows best practices)
- Efficiency (optimal time/space complexity)

### I can't get the tests to pass, what should I do?

1. Read the requirements carefully
2. Check the error messages - they often tell you what's wrong
3. Use the hint system (progressive hints available)
4. Review related lesson material
5. Take a break and come back fresh
6. Ask for help in the community

### Can I see the solution?

Yes, but only after:
- You've passed all tests, OR
- You've exhausted your attempts

We encourage you to try thoroughly before viewing the solution!

### Can I use external libraries?

Only pre-approved libraries are available. The exercise will tell you what's available.

### What languages are supported?

- JavaScript
- TypeScript
- Java

The language is specified per exercise.

## Progress & Achievements

### How is progress calculated?

Progress is calculated based on:
- Lessons completed
- Quizzes passed (score ≥ 70%)
- Exercises solved
- Flashcards mastered

Each component contributes to your overall completion percentage.

### What counts as "completing" a lesson?

Clicking the "Mark as Complete" button at the end of the lesson. We trust you to mark lessons complete when you've genuinely finished them.

### What are streaks?

A streak is consecutive days you've used the platform. To maintain a streak, you must complete at least 15 minutes of activity (lessons, reviews, exercises) per day.

### What are achievements?

Achievements are badges earned for milestones like:
- Completing your first lesson
- Maintaining a 7-day streak
- Reviewing 100 flashcards
- Passing all quizzes in a module

### Do achievements give any benefits?

Currently, achievements are for motivation and gamification. Future updates may add rewards like themes, avatars, or certificates.

### Can I export my progress data?

Yes! Go to Settings → Data Management → Export Progress. You can download your data as JSON or CSV.

## Offline Mode & PWA

### Does the platform work offline?

Yes! The platform is a Progressive Web App (PWA) with offline support for:
- Reading lessons (if previously viewed)
- Reviewing flashcards
- Viewing progress
- Writing code (submit when online)

### What doesn't work offline?

- Loading new lessons (first time)
- Submitting quizzes
- Submitting exercises for grading
- Viewing solutions
- Real-time features (leaderboards, etc.)

### How do I install the PWA?

**Desktop:**
- Chrome/Edge: Look for install icon in address bar
- Or use browser menu → "Install [app name]"

**iOS:**
- Safari → Share button → "Add to Home Screen"

**Android:**
- Chrome → Menu → "Add to Home screen"

### How does syncing work?

When you come back online:
- Your offline changes automatically sync to the server
- Progress updates, flashcard reviews, and notes all sync
- Conflicts are rare, but the app will notify you if any occur

### How much storage does offline mode use?

Typically 50-200 MB depending on how much content you've cached. You can view and manage storage in Settings → Offline Data.

## Technical Issues

### The site won't load

1. Check your internet connection
2. Try refreshing the page (Cmd/Ctrl + R)
3. Clear browser cache and cookies
4. Try a different browser
5. Disable browser extensions (ad blockers can interfere)
6. Try incognito/private mode

### Flashcards aren't showing

- Check your review settings (daily limit)
- Verify there are cards due (check calendar)
- Try syncing manually (click sync icon)
- Clear offline cache in Settings

### Code editor isn't working

- Ensure JavaScript is enabled
- Try a different browser
- Update to the latest browser version
- Check if any extensions are blocking the editor

### My progress isn't saving

- Ensure you're logged in
- Check your internet connection
- Wait for the sync icon to confirm save
- Try manual sync if needed

### I found a bug, how do I report it?

Please report bugs:
- GitHub Issues: [repository link]
- Email: support@playwright-selenium-learning.com
- Include: Browser version, OS, steps to reproduce, screenshots
